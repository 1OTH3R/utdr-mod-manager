# UTDR Mod Manager - v1.1!

A genuinely simple manager for Undertale and Deltarune mods (with some fangame support as well)!
*Designed to make setup and usage as easy as possible.*

---

## Installation Guide

1. Run the installer executable file
2. Enjoy!

---

## Features

* An official and easy-to-use Windows app installer!
* Per-mod saves! ✨
* A stable application (yippee)!
* Launching, playing, and deleting mods.
* Local mod storage.
* Metadata and images.
* File viewing.
* Mediocre fangame support (eh, it's okay).
* And (maybe) more!

---

## A quick run-down

This mod manager is relatively no-nonsense, but just in case...

### Toolbar (top right)

* Click the floppy disk (💾) to add mods
* Click the arrow/triangle (▶️) to play mods
* Click the folder (📂) to open a mod's files
* Click the rightmost icon (💣) to delete a mod

### Navigation (middle left)

* Click on mods to select them and view their data (ie: images, descriptions, authors, ect.)!

## Q&A

**Q: Does this manager hold pre-modded games, or does it take and apply .xdelta files?**
A: As of now, it only holds pre-modded games, but applying one .xdelta file to a game is in the works! Do keep in mind that I'm still relatively new to JavaScript coding as a whole (this project was my intro to JavaScript), so multi-mod support might be awhile, sorry.

**Q: Where can I go if I have unresolved bugs?**
A: I'd prefer if you reported your bug in the [Issues](https://github.com/1OTH3R/utdr-mod-manager/issues) tab on GitHub, but if you don't want to make a GitHub account, you can also use the one here in GameBanana! Just do keep in mind that there will be quite a bit of questions asked to ensure I can vanquish the bug in its entirety!

**Q:** **I want to have my voice heard!** **Where can I put my suggestions?**
A: Just like with bugs, suggestions can go in the [Issues](https://github.com/1OTH3R/utdr-mod-manager/issues) tab on GitHub, but once again, if you don't want to make a GitHub account, you can also use the one here in GameBanana!

**Q: Why didn't you just update your previous mod manager's page? Why create a new page?**
A: When I made that first "mod manager", it was built on Python, then a conglomerate of a billion different file types strung together, so it wasn't exactly great. However, I have since abandoned that project and switched to a completely different motive, architecture, workload, and so much more. These projects are important to me, but I needed to separate it from my previous project, because they're just so wildly different.
